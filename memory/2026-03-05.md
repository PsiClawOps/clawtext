## App detection reports a game at boot (Contractors Showdown) — detection should be empty

On SteamVR boot (before any game launch), RGCS reports the detected application as "Contractors Showdown" instead of empty. Likely cause is scanning SteamVR logs too far back or incorrect detection logic; detection should be blank until an app actually starts. This is a bug to fix in the detection logic.

## Functional test procedure for validating smoothing (including MaxDiag requirement)

Validated test steps the user uses: 1) press <mode> button; 2) turn head quickly; 3) raise hand and check controller laser is enabled; 4) press disable to compare. Iterate this for recommended/strong/maxdiag. The user requires that MaxDiag apply excessive strength (making smoothing unusable) as a validation; note that MaxDiag has sometimes failed to show the expected extreme effect.
