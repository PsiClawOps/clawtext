## RGCS v1.2.335 — Smoothing Had No Discernible Effect (Bug + Fix)

**Root cause:** `effectiveMinCutoff = configuredMinCutoff * 0.01^strength` curve was designed
for strength in 0–1. After rescaling physical max to 0.16 (ctrl) / 0.02 (HMD), DiagMax only
put 0.16 into the exponent → `1.5 * 0.01^0.16 = 0.747Hz` (barely any smoothing).
At 50% (0.08 ctrl) it computed `~0.87Hz` — effectively nothing.

**Fix (commit 1024d36):** Normalize physical strength by per-device max before applying curve:
```cpp
float ctrlNormalized = ctrlStrength / CONTROLLER_MAX_PHYSICAL;  // 0.16/0.16 = 1.0 at DiagMax
float cutoff = configuredMinCutoff * pow(0.01f, ctrlNormalized);  // → ~0.015Hz as intended
```
Same fix for HMD path using HMD_MAX_PHYSICAL.

**Expected after fix:** DiagMax controller → `~0.015Hz` (heavy), Recommended (50%) → `~0.12Hz` (moderate).
Smoothing should now be perceptible at all preset levels.

2. **DiagMax beta=1.5 (commit e65d44d):** Velocity-adaptive beta meant the OneEuro adaptive cutoff formula `cutoff = minCutoff + beta * |velocity|` raised the cutoff with any motion. At slow aim (~0.1 m/s): `0.05 + 1.5 * 0.1 = 0.2Hz` — filter transparent. Fixed: DiagMax now uses `beta=0.0` (fully non-adaptive). No velocity bailout. Will always be perceptible.
